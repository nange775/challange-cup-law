# 多格式证据导入功能使用说明

## 功能概述

系统现在提供**统一的多格式证据导入**功能，支持以下文件格式：

| 格式类型 | 扩展名 | 处理方式 |
|---------|--------|----------|
| **Excel** | .xls, .xlsx | 表格数据解析（含财付通数据） |
| **PDF** | .pdf | 文字提取 |
| **Word** | .doc, .docx | 文档解析 |
| **文本** | .txt | 直接读取 |
| **图片** | .jpg, .jpeg, .png, .tif, .tiff | **OCR文字识别** |

## 使用步骤

### 1. 进入数据导入页面
点击左侧导航栏的 **"数据导入"**

### 2. 进入多格式证据导入页面
系统默认显示 **"数据导入"** 页面，已整合所有导入功能

### 3. 填写基本信息

| 字段 | 说明 | 示例 |
|------|------|------|
| **案件ID** | 证据所属案件编号 | `CASE-2024-001` |
| **证据类型** | 选择或自动识别 | 讯问笔录/合同/流水等 |
| **证据描述** | 补充说明（可选） | 来源、提取时间等 |

### 4. 上传文件
- 点击 **"选择文件"** 按钮
- 支持同时选择**多个文件**
- 支持拖拽上传

### 5. 开始导入
点击 **"导入证据"** 按钮，系统将：
1. 自动识别文件格式
2. 提取文字内容（图片自动OCR）
3. 识别证据类型
4. 提取关键实体（人名、金额、日期等）
5. 关联到对应人员

### 6. 查看结果
导入完成后显示结果表格：
- **文件名** - 上传的文件名
- **状态** - 成功/重复/失败
- **识别类型** - 自动识别的证据类型
- **提取实体** - 识别出的人名等关键信息

## 财付通数据导入说明

原有的财付通数据导入功能已合并到多格式证据导入中：

1. **准备文件**：`TenpayTrades.xls`（交易明细）
2. **选择类型**：证据类型选择 **"交易流水"** 或保持 **"自动识别"**
3. **上传导入**：系统将自动解析财付通格式并入库

## OCR功能说明

上传图片（JPG/PNG/TIFF等）时，系统将：

1. **自动进行OCR文字识别**
2. **提取中文和英文内容**
3. **识别关键实体**：
   - 姓名（如：张三、李四）
   - 金额（如：10000元、5万元）
   - 日期（如：2024年1月15日）
   - 身份证号、手机号等

### OCR依赖安装

如需使用OCR功能，请安装：

```bash
# 1. 安装 Tesseract OCR 引擎

# Ubuntu/Debian:
sudo apt-get install tesseract-ocr tesseract-ocr-chi-sim

# Windows:
# 下载: https://github.com/UB-Mannheim/tesseract/wiki
# 安装时选择 Chinese (Simplified) 语言包

# Mac:
brew install tesseract tesseract-lang

# 2. 安装 Python 依赖
pip install pytesseract Pillow
```

## 常见问题

### Q1: 导入失败怎么办？
A: 检查：
- 文件格式是否支持
- 文件是否损坏
- 案件ID是否填写
- 后端日志错误信息

### Q2: OCR识别率低怎么办？
A: 
- 确保图片清晰度足够
- 文字方向要正向
- 避免过度倾斜或模糊
- 安装Tesseract中文语言包

### Q3: 财付通数据如何导入？
A: 像普通Excel一样上传，系统会自动识别财付通格式特征

## 技术架构

```
前端 (Vue.js)
    ↓ 上传文件 + 案件ID + 类型
后端 API (/api/evidence/import)
    ↓ 根据文件扩展名选择
格式解析器 (Parsers)
    ├── ExcelParser (.xls/.xlsx)
    ├── PDFParser (.pdf)
    ├── WordParser (.doc/.docx)
    ├── ImageParser (.jpg/.png/.tiff) ← OCR
    └── TextParser (.txt)
    ↓ 提取内容
AI识别模块
    ↓ 自动识别证据类型
数据库
    ├── evidence_meta (元数据)
    ├── transactions (交易流水)
    ├── interrogation_records (讯问笔录)
    └── ... (其他专门表)
```
